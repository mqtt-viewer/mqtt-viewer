# Client logs (per-connection MQTT library logs)

Issue: https://github.com/mqtt-viewer/mqtt-viewer/issues/4

## Goal

Give each connection an accessible, terminal-styled view of the diagnostic
output the MQTT client libraries emit while talking to the broker
(connect / reconnect / disconnect / subscribe / ping / errors). Today those
library loggers are commented out and nothing is captured. We capture them
per-connection into a bounded in-memory buffer **and** a durable rotating text
file, and surface them via a **"View logs"** item in the existing dropdown under
the connection name (top-left), opening a monospace dialog with live updates,
copy, clear, text filter, and level filters.

## Non-goals

- No changes to the app-wide/global slog logging (`backend/logging`) or its
  `log.txt`. This is a separate, connection-scoped facility.
- No remote log shipping, no export formats beyond copy-to-clipboard.
- No log viewer for anything other than MQTT client-library/connection logs.
- Not adding a general in-app console for non-MQTT subsystems.

## Design

### Log entry shape (shared, Go → TS via bindings)

```go
// backend/mqtt/log_store.go
type LogEntry struct {
    TimestampMs int64  `json:"timestampMs"` // wall-clock ms
    Level       string `json:"level"`       // "debug" | "info" | "warn" | "error"
    Message     string `json:"message"`
}
```

### What is captured, and the debug toggle

Two sources feed a connection's `LogStore`:

1. **Always-on lifecycle + errors (per-connection, both v3 & v5).** Emitted from
   inside `MqttManager` at the points it already observes state — primarily
   `SetConnectionState` (connecting / connected / reconnecting+cause /
   disconnected+cause) plus connect-time and subscribe-time errors in
   `connectV5`/`connectV3`. Low volume; these are the essential "client logs" and
   are accurately attributed to the connection regardless of MQTT version.

2. **Verbose library debug (opt-in via the per-connection toggle).**
   - **v5:** the autopaho `Debug`, `PahoDebug`, `PahoErrors` loggers are wired to
     the connection's `LogStore` (debug/errors respectively). They are **always
     assigned**, but the store **drops `debug`-level writes when the toggle is
     off** — so toggling takes effect instantly, with no reconnect required.
   - **v3:** because paho v3 loggers are global package vars, verbose debug can
     only be captured globally. A package-level dispatcher (installed once) fans
     the global v3 log lines out to **every v3 connection that currently has the
     toggle on**. Accurately per-connection attribution is not possible for v3
     verbose lines — documented limitation (see Edge cases). v3 lifecycle/errors
     (source #1) remain accurate because they come through `SetConnectionState`.

The toggle is persisted per connection (`Connection.DebugLoggingEnabled`), and
its live value is mirrored onto the `LogStore` so gating needs no DB read.

### LogStore (new: `backend/mqtt/log_store.go`)

A first-class member of `MqttManager` (alongside `MessageBuffer`), holding:

- `entries []LogEntry` ring buffer, cap **2000** (drop-oldest), guarded by a mutex.
- `debugEnabled atomic.Bool`.
- A lazy **lumberjack** writer to the connection's file (see Persistence);
  `nil`/no-op until initialised.
- A pending-batch slice + ticker (mirrors `MessageBuffer`) that drains new
  entries to an `onBatch func([]LogEntry)` callback (set by the app layer to emit
  the Wails event). Interval **300ms** (reuse `MQTT_BUFFER_EMIT_INTERVAL`-style
  cadence) to coalesce chatty debug output.

Methods:
- `log(level, msg)` — timestamp, append to ring (evict oldest past cap), append
  to pending batch, and write one line to the file. `debug` level is a **no-op
  when `debugEnabled` is false** (nothing buffered, batched, or filed).
- Convenience `Info/Warn/Error/Debug(msg string)`.
- `Snapshot() []LogEntry` — copy of the ring (for the RPC getter).
- `Clear()` — empties the ring and **truncates the file(s)**.
- `SetDebugEnabled(bool)`.
- lifecycle: `startEmitting(onBatch)` / `stopEmitting()`; `initFile(path)`;
  `close()`.

**paho adapters** (same file): a small type implementing `Println`/`Printf`
(satisfies both `paho.Logger` and `mqttV3.Logger`) that formats args and calls a
bound sink func (`store.Debug` or `store.Error`).

### Persistence (durable text file)

- Path: `<ResourcePath>/logs/connections/conn-<id>.txt`.
- **lumberjack** rotation: `MaxSize: 5` MB, `MaxBackups: 3` (a connection's log
  is less critical than the app log; keep disk modest).
- Written whenever `log()` runs (i.e. always-on lifecycle/errors even with debug
  off; verbose debug added when the toggle is on).
- Line format in the file: `2006-01-02T15:04:05.000  LEVEL  message`.
- "Clear" truncates the active file (and removes rotated backups best-effort).

### MqttManager integration (`backend/mqtt/manager.go`)

- Add `LogStore *LogStore` member; construct an in-memory store in
  `NewMqttManager` (no file/emit yet — keeps the 2-arg constructor and test
  caller intact).
- Add setters (called from the app layer):
  `InitLogging(filePath string, debugEnabled bool, onBatch func([]LogEntry))`,
  `SetDebugLoggingEnabled(bool)`, `GetLogs() []LogEntry`, `ClearLogs()`.
- In `SetConnectionState`, after updating state, mirror the transition to
  `LogStore` (`Info` for normal, `Error` when `reason != nil`), reusing the same
  strings it already slogs.
- In `connectV5`: assign `config.Debug`/`config.PahoDebug` → debug adapter,
  `config.PahoErrors` → error adapter; log connect/subscribe errors to the store.
- In `connectV3`: on connect, register this connection's store in the global v3
  dispatcher if debug is enabled (and unregister on disconnect); log
  connect/subscribe errors to the store. Install the global v3 loggers once (see
  below).

### v3 global dispatcher (`backend/mqtt/log_store.go`)

- A package-level registry: `map[uint]*LogStore` (connId → store) + mutex, of
  v3 connections with debug enabled.
- `installV3GlobalLoggers()` (guarded by `sync.Once`) sets
  `mqttV3.DEBUG/WARN/ERROR/CRITICAL` to adapters that broadcast to all registered
  stores at the mapped level.
- `registerV3Debug(connId, store)` / `unregisterV3Debug(connId)` — called by the
  manager when the toggle flips or the v3 connection ends.

### App layer (`backend/app/`)

- `startup.go` `createAppConnectionFromConnectionModel`: after building the
  manager, call `mqttManager.InitLogging(logPath, conn.DebugLoggingEnabled,
  onLogBatch)` where:
  - `logPath = filepath.Join(a.resourcePath(), "logs", "connections",
    fmt.Sprintf("conn-%d.txt", conn.ID))` (create dir with `os.MkdirAll`).
    Reuse the existing resource-path accessor the app already holds (same source
    `backend/logging` uses); if none is exposed on `App`, thread it in from where
    `InitLogger` is set up.
  - `onLogBatch = func(entries []mqtt.LogEntry) { if a.Mode != AppModes.Test {
    a.EventRuntime.EventsEmit(connEvents.MqttLogs, entries) } }`.
  - Guard `DebugLoggingEnabled` nil (pointer/bool default false).
- New bound methods in `backend/app/mqtt.go`:
  - `GetConnectionLogs(connId uint) ([]mqtt.LogEntry, error)` → `manager.GetLogs()`.
  - `ClearConnectionLogs(connId uint) error` → `manager.ClearLogs()`.
  - `SetConnectionDebugLogging(connId uint, enabled bool) error` → persist
    `debug_logging_enabled` on the `connections` row **and**
    `manager.SetDebugLoggingEnabled(enabled)` (which also (de)registers the v3
    dispatcher when a v3 connection is active).

### Events (`events/connections.go`)

- Add `const MQTT_LOGS = "server:mqtt-logs"`.
- Add `MqttLogs string \`json:"mqttLogs"\`` to `ConnectionEventsSet` and populate
  it (`MQTT_LOGS + ":" + connId`) in `GetConnectionEventsSet`.

### Model + migration

- `backend/models/models.go` `Connection`: add

### Frontend

- **`ConnectionLogsDialog.svelte`** (new, beside `ConnectionDetailsDialog.svelte`):
  props `isOpen: Writable<boolean>`, `connection`. On open: call
  `GetConnectionLogs(id)` for the snapshot, then `Events.On(
  connection.eventSet.mqttLogs, …)` to append batches; unsubscribe + clear local
  state on close. Terminal look: `startEmpty` Dialog wrapping a dark

## Edge cases

- **Debug toggle latency:** v5 takes effect instantly (sink-gated). v3 verbose
  debug only starts/stops as connections register/unregister; lifecycle logs are
  unaffected. Documented.
- **Debug-off connections ignore verbose debug (guaranteed).** A connection
  whose toggle is off never captures library debug lines: v5 drops `debug`-level
  writes at the sink (`log()` early-returns when `debugEnabled` is false), and v3
  connections are only added to the global dispatcher registry via
  `registerV3Debug` when their toggle is on — a debug-off connection is never in
  the registry, so it receives none of the global v3 debug output. Debug-off
  connections still capture their always-on lifecycle/error lines.
- **v3 verbose attribution:** verbose v3 debug lines broadcast to *all*
  debug-enabled v3 connections (global library loggers). If a user runs multiple
  v3 connections **with debug on** simultaneously, their verbose lines interleave
  across those connections' logs (this only affects connections that opted in —
  see the guarantee above). Lifecycle/error lines remain correctly
  per-connection. Acceptable, documented; v5 (the default) is unaffected.
- **Dialog opened while disconnected:** snapshot returns whatever is buffered
  (possibly empty or prior-session file not loaded into RAM — RAM ring is
  session-scoped). Empty state shows a muted "No logs yet" message.
- **File vs RAM divergence:** the file is the durable superset (survives
  restart, rotated); the in-RAM ring (2000) backs the live dialog. The dialog
  shows RAM; the file is for post-hoc inspection. Not surfacing "open file" in
  UI for v1 (out of scope) — the path is deterministic.
- **Clear:** empties RAM ring and truncates the file; the open dialog clears its
  local list. Does not affect other connections.
- **Chatty brokers:** bounded ring + 300ms batched emit + opt-in verbose keep
  memory and IPC bounded. Always-on volume is only lifecycle/errors.
- **Nil `DebugLoggingEnabled`:** treated as `false`.
- **Test mode:** no event emission (mirror existing `a.Mode != AppModes.Test`
  guards); file logging still exercised by unit tests via a temp dir.

## Decisions log

- **Per-connection LogStore (not a global-slog hook).** Explicit, decoupled from
  app logging, O(1) per line, mirrors `MessageBuffer` — consistent with the
  codebase. Rationale over hooking `slogmulti.Fanout`: avoids O(connections) work
  on every global log line and keeps "client logs" cleanly separated from
  internal app logs.
- **Always-on lifecycle + opt-in verbose** (slight expansion of the literal
  "toggle off = capture nothing"), approved by the user at direction review: the
  dialog is never empty and useful without enabling the firehose.
- **v5 sink-gated debug** (always-assigned loggers, drop at sink when off) so the
  toggle is instant with no reconnect. v5 is the default path.
- **v3 verbose debug is global/broadcast** due to paho v3's package-level
  loggers; lifecycle stays per-connection. Chosen over dropping v3 verbose
  entirely (still useful) and over faking attribution.
- **Durable rotating text file via lumberjack** (user asked for on-disk logs):
  reuses the existing dependency + rotation pattern; human-readable and
  shareable. 5 MB × 3 backups per connection.
- **2000-line RAM ring** for the live view; **300ms batched** event emit to
  coalesce chatty debug.
- **New DB column + migration** for the persisted per-connection toggle, matching
  the existing SQL-migration + `atlas migrate hash` workflow.

### Implementation deviations (local, design unchanged)

- **Bindings hand-edited, not full-regen.** The installed `wails3` CLI
  (v3.0.0-alpha.98) emits a different, incompatible binding *style*
  (`$CancellablePromise` + `.then(createType)`) than the committed bindings
  (`Promise & { cancel() }`, generated by the `-tui` variant pinned in go.mod).
  A full regen rewrote every binding file. To keep the PR minimal and consistent
  with the committed toolchain output, only the four feature-relevant binding
  files were edited additively, using the stable name-based method IDs the
  generator assigns (`GetConnectionLogs` 226016115, `ClearConnectionLogs`
  1243746974, `SetConnectionDebugLogging` 1317832316).
- **Removed unused `backend/mqtt/logger.go`.** Its `MqttLogger` (unused, loggers
  were commented out) is superseded by the paho adapters in `log_store.go`.
- **Not verified here:** live end-to-end against a real broker (needs the Wails
