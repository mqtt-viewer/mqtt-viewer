# Design-System Sync Workflow

This repo uses Storybook plus colocated component specs to keep the Svelte codebase ready for future Figma sync work.

## Source Of Truth

Each component under `frontend/src/components` and `frontend/src/views` has:

- `<Component>.svelte` for implementation
- `<Component>.stories.svelte` for rendered Storybook coverage
- `<Component>.spec.json` for the machine-readable design contract

The colocated spec is the file a human or implementation agent edits. The aggregate index at `frontend/src/design-system/component-index.json` is generated by `pnpm ds:registry` and should not be hand-edited.

## Local Commands

Run these from `frontend/`:

```bash
pnpm storybook
pnpm build-storybook
pnpm ds:tokens
pnpm ds:registry
pnpm ds:validate
pnpm ds:make-spec Button
```

`pnpm ds:validate` regenerates `design-tokens.json`, `component-index.json`, and `COMPONENT_CHECKLIST.md`, then validates schemas, story coverage, token references, and sync-key consistency.

## Storybook Rules

Stories are colocated and use `@storybook/addon-svelte-csf`:

- `title` starts with `Primitives/`, `Components/`, or `Views/` to match the spec tier.
- `tags: ["autodocs"]` is required.
- `parameters.design = { type: "figma", url: "" }` is present even before Figma exists.
- Wails bindings are mocked through `.storybook/mocks/wailsjs`.
- Shared mock data lives in `frontend/src/stories/fixtures.ts`.

## Figma Is Not Linked Yet

Current component specs use `status: "story-only"` and empty Figma fields. When Figma exists, update each relevant colocated spec:

```json
{
  "status": "figma-linked",
  "figma": {
    "fileKey": "...",
    "nodeId": "...",
    "url": "...",
    "lastSyncedHash": "sha256:...",
    "lastSyncedAt": "2026-05-26T00:00:00Z"
  }
}
```

For primitives, map variant props in `props[]` and list consumed design tokens in `tokens[]`.

## Figma -> AI -> Code Loop

1. Designer edits the Figma component or frame. The Figma component name should match the code component name where possible.
2. Drift agent reads `component-index.json` or colocated specs, pulls linked Figma nodes, recomputes the structural hash, and compares Figma variant properties/tokens to `props[]` and `tokens[]`.
3. Drift agent writes a handover doc in `frontend/src/design-system/changes/<date>-<component>.md` with concrete code/spec/story changes.
4. Implementation agent applies the handover, updates the colocated spec, updates the story if needed, and runs `pnpm ds:validate`.
5. Verification uses `pnpm storybook`, `pnpm build-storybook`, and Storybook MCP screenshots/tests when configured.

## Storybook MCP

Use `mcpland/storybook-mcp` for renderer-agnostic Storybook read access. Build or serve Storybook first, then point `STORYBOOK_URL` at the generated index:

```json
{
  "mcpServers": {
    "storybook": {
      "command": "npx",
      "args": ["-y", "storybook-mcp"],
      "env": {
        "STORYBOOK_URL": "http://localhost:6006/index.json"
      }
    }
  }
}
```

Storybook MCP describes rendered reality. The colocated specs describe Figma sync state, tier, tokens, and dependency rules. Keep both current.
